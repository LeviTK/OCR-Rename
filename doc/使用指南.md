# 票据条形码 / 二维码批量识别工具 — 使用指南

## 📁 目录结构

```
003_HD/
├── 001-Pic/       ← 放入待识别的图片
├── 识别正确/      ← 识别成功后自动归档（以码值重命名）
├── 未识别/        ← 无法识别的图片自动归档
├── src/           ← 程序源码
└── .venv/         ← Python 虚拟环境
```

## 🚀 快速开始

### 1. 放入图片

将待识别的票据图片（jpg / png / bmp 等）放入 **`001-Pic/`** 文件夹。

### 2. 打开终端，进入项目目录

```bash
cd /path/to/003_HD
```

### 3. 运行识别

```bash
DYLD_LIBRARY_PATH=/opt/homebrew/lib .venv/bin/python -m src
```

程序会自动：
- 扫描 `001-Pic/` 中的所有图片
- 同时识别每张图片上的条形码和二维码，只接受纯数字结果
- 若条形码和二维码识别出相同数字，则优先采用这个共同值
- 若只识别出单侧数字结果，则采用分数最高的数字值
- 识别成功 → 以最终数字码值重命名，移动到 **`识别正确/`**
- 识别失败 → 保留原名，移动到 **`未识别/`**

### 4. 预览模式（不移动文件）

```bash
DYLD_LIBRARY_PATH=/opt/homebrew/lib .venv/bin/python -m src --dry-run
```

只显示识别结果，不做任何文件操作，适合先确认再执行。

## ⚙️ 首次安装

如果 `.venv` 不存在，需要先初始化环境：

```bash
# 1. 安装系统依赖（仅需一次）
brew install python zbar tesseract

# 2. 创建虚拟环境并安装 Python 依赖
cd /path/to/003_HD
python3 -m venv .venv
.venv/bin/pip install pillow opencv-python numpy pyzbar zxing-cpp
```

## 💡 提示

- **支持格式**：jpg、jpeg、png、webp、bmp、tif、tiff
- **业务前提**：条形码和二维码的业务值应一致，且均为纯数字
- **实时进度**：第一轮固定 3 步，第二轮固定 5 步，终端会实时显示当前做到哪一步
- **二维码扫描策略**：不再预先判断图片里是否存在二维码，所有图片都会默认进入完整二维码深扫，避免把“实际有二维码”的图片误判成无码
- **结果选择**：优先采用“条形码和二维码共同命中的数字值”，否则采用单侧最高分数字值
- **重名处理**：如果码值相同，自动加后缀 `_1`、`_2` 避免覆盖
- **识别速度**：清晰图片 < 1秒，模糊/变形图片可能需要 10-30 秒（深度扫描）
